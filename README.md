# Medicare Geographic Inequity Analysis

Detects geographic over/underpayment in Medicare physician reimbursement using a
Gradient Boosting model trained on the 2021 CMS dataset, with geography deliberately
excluded as a feature. Residuals (actual minus predicted) expose where the payment
system systematically favors or penalizes providers based solely on location.

---

## Setup

```bash
pip install xgboost scikit-learn pandas numpy matplotlib seaborn pyarrow
```

---

## Data

Download the 2021 CMS file from:

> https://data.cms.gov/provider-summary-by-type-of-service/medicare-physician-other-practitioners/medicare-physician-other-practitioners-by-provider-and-service

Place the CSV in the `data/` folder. The expected filename is:

```
data/MUP_PHY_R24_P05_V10_D21_Prov_Svc.csv
```

If CMS renames the file, update `RAW_DATA_PATH` in `01_load_and_preprocess.py`.

---

## Run the full pipeline

```bash
python run_pipeline.py
```

Or run individual steps:

```bash
python 01_load_and_preprocess.py   # clean & encode CMS data → data/processed_cms_2021.parquet
python 02_train_model.py           # train XGBoost → models/gbm_no_geo.pkl + data/predictions.parquet
python 03_residual_analysis.py     # aggregate residuals → outputs/*.csv
python 04_visualize.py             # figures → outputs/figures/*.png

# Run only steps 3 and 4 (if model already trained):
python run_pipeline.py --steps 3 4
```

## Methodology

| Step | What it does |

| **Preprocess** | Load CMS CSV, compute per-service payment, encode HCPCS/provider type, add rural flag from RUCA code |
| **Train** | XGBoost regressor on `[hcpcs_code, provider_type, place_of_service, total_services, total_beneficiaries]` — **no geographic features** |
| **Residuals** | `residual = actual_payment_per_service − predicted_payment_per_service`. Aggregate by state and rural/urban; t-test each state against zero |
| **Visualize** | State bar chart, rural/urban comparison, residual distributions, top underpaid HCPCS codes, rural-% scatter |

**Why exclude geography from the model?**
The goal is to learn what a service *should* cost based purely on what it is — then
measure where the system pays more or less than that. Including geography would let
the model absorb the very inequity we're trying to detect.

---

## Key Output: `outputs/state_residuals.csv`

| Column | Meaning |

| `mean_residual` | Average (actual − predicted) $/service. Negative = underpaid. |
| `pct_rural` | Share of provider records flagged as rural (RUCA ≥ 4) |
| `p_value` | One-sample t-test p-value against zero |
| `significant_underpay` | True if mean_residual < 0 and p < 0.05 |
